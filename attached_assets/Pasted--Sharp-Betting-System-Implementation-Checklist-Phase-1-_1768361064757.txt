### ☐ 3.4 Compare Old vs New
Run this query:
```sql
SELECT 
    away_team || ' @ ' || home_team as matchup,
    edge as raw_edge,
    true_edge,
    edge - true_edge as difference,
    CASE 
        WHEN edge >= 8.0 THEN 'Old System: YES'
        ELSE 'Old System: NO'
    END as old_qualified,
    CASE 
        WHEN is_qualified THEN 'New System: YES'
        ELSE 'New System: NO'
    END as new_qualified
FROM game
WHERE date = CURRENT_DATE
  AND edge IS NOT NULL
ORDER BY edge DESC
LIMIT 10;
```

You should see cases where:
- Old system qualified (edge >= 8.0)
- New system rejected (true_edge < 3.5 OR negative EV OR low history)

## Common Issues & Solutions

### ❌ Issue: ImportError on sharp_betting
**Solution:** Make sure `sharp_betting.py` is in the same directory as your main app file

### ❌ Issue: No picks qualifying at all
**Solution:** Check if `history_qualified` is populated. Run `update_game_historical_data()` first

### ❌ Issue: Database error on new columns
**Solution:** Make sure migration ran successfully. Check: `\d game` in psql

### ❌ Issue: True edge equals raw edge
**Solution:** Verify vig removal is running. Check `vig_percentage` column has values

### ❌ Issue: All picks show "ELITE" confidence
**Solution:** Check that EV and history filters are enabled in qualification logic

## Success Criteria ✅

You'll know it's working when:

1. **Fewer Picks** - Should see 30-40% fewer qualified picks
2. **Different Edges** - `true_edge` should be 40-60% of `raw_edge`
3. **EV Filtering** - No picks with negative EV qualify
4. **Kelly Sizing** - Bet sizes range from 0.5% to 5% (most around 1-2%)
5. **Better Win Rate** - After 50+ picks, win rate should be 60%+
6. **Positive CLV** - When you implement tracking, average CLV should be +0.5 points
