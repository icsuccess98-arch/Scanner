Fix database sync issues between preview (dev) and deployed (production):

PROBLEM IDENTIFIED:
Preview shows picks → Deploy → Production shows different/no picks

ROOT CAUSES:
1. Preview uses DEV database
2. Production uses PROD database
3. Databases are NOT synced
4. Picks saved to dev don't appear in prod

═══════════════════════════════════════════════════════════════════════

DIAGNOSIS CHECKLIST:

Check your environment variables:
□ Preview: DATABASE_URL = dev database
□ Production: DATABASE_URL = prod database
□ Are they pointing to SAME database?

If different databases:
□ Dev has picks from testing
□ Prod is empty or has old data
□ Result: Mismatch after deployment

═══════════════════════════════════════════════════════════════════════

SOLUTION 1: USE SINGLE DATABASE (Recommended)

Make preview and production use THE SAME database:

1. UPDATE ENVIRONMENT VARIABLES:
   In your deployment platform (Replit, Vercel, etc.):
   
   Preview Environment:
   DATABASE_URL=postgresql://user:pass@host/prod_db
   
   Production Environment:
   DATABASE_URL=postgresql://user:pass@host/prod_db
   
   ☝️ SAME database URL

2. VERIFY:
```python
   import os
   print(f"Database: {os.getenv('DATABASE_URL')}")
   # Should print SAME URL in both environments
```

3. RESTART BOTH:
   - Stop preview
   - Deploy to production
   - Start preview
   - Both should now show same data

SOLUTION 2: AUTO-SYNC DATABASES (If must keep separate)

If you MUST use separate dev/prod databases:

1. SYNC SCRIPT:
```python
   def sync_dev_to_prod():
       """
       Copy picks from dev to prod before deployment
       """
       # Connect to BOTH databases
       dev_db = connect_to_database(os.getenv('DEV_DATABASE_URL'))
       prod_db = connect_to_database(os.getenv('PROD_DATABASE_URL'))
       
       # Get all active picks from dev
       dev_picks = dev_db.query("""
           SELECT * FROM picks 
           WHERE status = 'ACTIVE' 
           AND game_date >= CURRENT_DATE
       """).fetchall()
       
       # Insert into prod (skip if exists)
       for pick in dev_picks:
           prod_db.execute("""
               INSERT INTO picks (...)
               VALUES (...)
               ON CONFLICT (game_id, pick_type) DO UPDATE SET
                   posted_line = EXCLUDED.posted_line,
                   posted_edge = EXCLUDED.posted_edge,
                   confidence = EXCLUDED.confidence
           """)
       
       prod_db.commit()
       print(f"✅ Synced {len(dev_picks)} picks to production")
```

2. RUN BEFORE DEPLOY:
```bash
   # In your deploy script
   python sync_databases.py
   replit deploy
```

3. AUTOMATED:
```python
   # Add to your app startup
   if os.getenv('ENVIRONMENT') == 'production':
       # On first production startup
       if not picks_exist():
           sync_from_dev()

SOLUTION 5: CHECK WHEN PICKS ARE SAVED

   def save_pick_to_db(pick):
       logger.info(f"💾 Saving pick to database: {pick['game']}")
       logger.info(f"   Database: {os.getenv('DATABASE_URL')[:30]}...")
       
       result = db.picks.insert(pick)
       
       logger.info(f"✅ Saved pick ID: {result.id}")
       
       # Verify it was saved
       verify = db.picks.find_one({'id': result.id})
       if verify:
           logger.info(f"✅ Verified pick exists in database")
       else:
           logger.error(f"❌ Pick NOT found after save!")
```
